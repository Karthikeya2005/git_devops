🐙 Learning GitHub

Your beginner's guide to version control with Git & GitHub.

📌 Key Concepts

  • Repository (Repo) – A folder that stores your project and its history.
  • Commit – A saved snapshot of your changes.
  • Branch – A separate line of development.
  • Merge – Combining changes from one branch into another.
  • Pull Request (PR) – A request to merge your changes into the main branch.
  • Clone – Copying a remote repo to your local machine.
  • Fork – Your personal copy of someone else's repo.

⚙️ Basic Git Commands

Set up Git:

git config --global user.name "Your Name"
git config --global user.email "you@example.com"

Start a project:

git init
git clone https://github.com/username/repo.git

Track & save changes:

git status
git add .
git commit -m "Your commit message"

Push & pull:

git push origin main
git pull origin main

Branching:

git branch new-feature
git checkout new-feature
git merge new-feature

🔄 Typical GitHub Workflow

  1. Create or fork a repository
  2. Clone it to your local machine
  3. Create a new branch for your feature
  4. Make changes and commit them
  5. Push the branch to GitHub
  6. Open a Pull Request
  7. Review and merge the PR

💡 Tips for Beginners

Always write clear commit messages — future you will thank you!
Use .gitignore to avoid pushing unnecessary files like node_modules.
Never push directly to the main branch — use branches and PRs.
Use git status often to see what's going on in your repo.
